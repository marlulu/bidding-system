# 内部招标网站系统 - 项目交付说明

## 一、项目概述

本项目是一个功能完整的内部招标网站系统，采用前后端分离架构开发，实现了招标信息管理、供应商管理、用户权限管理等核心功能。系统通过权限控制机制，确保不同角色用户能够访问相应的功能模块和数据信息。

## 二、技术架构

### 2.1 前端技术栈

本系统前端采用现代化的 Vue 生态技术栈，具体包括：

-   **Vue3**: 采用组合式 API（Composition API）开发，提供更好的代码组织和逻辑复用能力
-   **Vite**: 新一代前端构建工具，提供极速的开发体验和优化的生产构建
-   **Pinia**: Vue 官方推荐的状态管理库，用于管理用户信息等全局状态
-   **Vue Router**: 官方路由管理器，实现单页应用的路由跳转和权限控制
-   **Element Plus**: 基于 Vue3 的组件库，提供丰富的 UI 组件

### 2.2 后端技术栈

后端基于 Spring Boot 生态构建，采用分层架构设计：

-   **Spring Boot 3.x**: 简化 Spring 应用开发的框架，提供自动配置和快速启动能力
-   **MyBatis-Plus**: MyBatis 的增强工具，简化 CRUD 操作，提供分页、逻辑删除等功能
-   **MySQL 8.0**: 关系型数据库，存储业务数据
-   **JWT**: JSON Web Token，用于无状态的用户认证
-   **Fastjson2**: 阿里巴巴开源的 JSON 处理库

## 三、功能模块

### 3.1 模块一：门户管理

门户管理模块是系统的核心功能区域，负责招标信息的发布和展示。该模块包含以下功能：

**功能1：招标信息发布**

系统支持四种类型的招标公告发布，分别是招标公告、变更公告、终止公告和中标结果公示。管理员可以创建、编辑和发布这些公告，每个公告包含标题、公告编号、项目名称、项目预算、投标截止时间、联系方式等完整信息。公告支持草稿状态保存，待内容完善后再正式发布。

**功能2：信息隐藏和显示的控制**

系统实现了细粒度的信息可见性控制机制。每个招标公告可以设置为"公开"或"受限"两种可见级别。当设置为"受限"时，管理员可以指定特定的供应商列表，只有被选中的供应商才能查看该公告的详细内容。这一机制通过在数据库中存储可见供应商 ID 列表（JSON 格式），并在后端 Service 层进行权限校验来实现。

**功能3：政策法规发布**

政策法规模块允许管理员发布与招标相关的政策文件、法律法规和流程指南。政策法规按照"政策"、"法规"、"指南"三个类别进行分类管理，便于供应商快速查找所需信息。每条政策法规支持富文本内容编辑，可以包含详细的说明和要求。

**功能4：通知公告系统**

系统提供了完善的通知推送机制，管理员可以创建系统通知、政策变动通知和公告提醒三种类型的通知。通知支持两种推送方式：全员推送和定向推送。定向推送允许管理员选择特定的供应商接收通知。系统还实现了通知已读/未读状态跟踪，用户可以在系统中查看未读通知数量。

### 3.2 模块二：供应商库管理

供应商库管理模块负责维护供应商的基本信息和资质档案。

**功能1：供应商信息管理**

系统提供了完整的供应商信息维护功能，包括公司名称、统一社会信用代码、法人代表、联系人信息、公司地址等基本资料。每个供应商都有独立的认证状态，包括"待审核"、"已认证"和"已拒绝"三种状态。管理员可以审核供应商的资质，并更新其认证状态。

**功能2：供应商分类**

系统支持按照行业分类、企业规模（小型、中型、大型）等维度对供应商进行分类管理。这些分类信息可以在招标公告发布时作为筛选条件，帮助管理员快速定位目标供应商群体。

### 3.3 模块三：用户管理与权限管理

用户管理模块实现了系统的访问控制和权限管理。

**功能1：用户管理**

系统支持两种用户角色：管理员（ADMIN）和供应商（SUPPLIER）。管理员拥有系统的完整管理权限，可以创建、编辑、删除用户，以及管理所有业务数据。供应商用户只能查看与自己相关的招标信息、政策法规和系统通知。每个用户账号都可以设置启用或禁用状态，管理员可以随时控制用户的访问权限。

**功能2：登录认证系统**

系统采用基于 JWT 的无状态认证机制。用户通过用户名和密码登录后，系统生成包含用户身份信息的 JWT Token，前端在后续的每次请求中携带该 Token 进行身份验证。Token 有效期为 24 小时，过期后需要重新登录。密码采用 MD5 加密存储，确保用户账号安全。

## 五、数据库设计

系统数据库共包含 7 张核心业务表，采用 InnoDB 存储引擎，字符集为 utf8mb4。

### 5.1 核心表说明

| 表名 | 说明 | 主要字段 |
|------|------|----------|
| sys_user | 用户表 | id, username, password, role, status, supplier_id |
| supplier | 供应商表 | id, company_name, company_code, industry, scale, status |
| bid_announcement | 招标公告表 | id, title, type, content, visibility_level, visible_supplier_ids, status |
| policy_regulation | 政策法规表 | id, title, category, content, status |
| system_notice | 系统通知表 | id, title, content, type, target_type, target_supplier_ids, status |
| notice_read_record | 通知阅读记录表 | id, notice_id, user_id, read_time |

### 5.2 权限控制设计

系统在数据库层面和应用层面实现了双重权限控制：

-   **数据库层面**: 通过 `visible_supplier_ids` 和 `target_supplier_ids` 字段（JSON 类型）存储可见范围
-   **应用层面**: 在 Service 层根据用户角色和关联的供应商 ID 进行数据过滤

## 六、API 接口说明

系统提供 RESTful 风格的 API 接口，所有接口均返回统一的 JSON 格式响应。

### 6.1 统一响应格式

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

### 6.2 主要接口列表

| 模块 | 接口路径 | 方法 | 说明 |
|------|----------|------|------|
| 认证 | /api/auth/login | POST | 用户登录 |
| 认证 | /api/auth/info | GET | 获取当前用户信息 |
| 用户 | /api/users | GET | 获取用户列表 |
| 用户 | /api/users | POST | 创建用户 |
| 供应商 | /api/suppliers | GET | 获取供应商列表 |
| 供应商 | /api/suppliers/{id} | GET | 获取供应商详情 |
| 招标公告 | /api/announcements | GET | 获取招标公告列表 |
| 招标公告 | /api/announcements/{id}/publish | PUT | 发布招标公告 |
| 政策法规 | /api/policies | GET | 获取政策法规列表 |
| 系统通知 | /api/notices | GET | 获取系统通知列表 |
| 系统通知 | /api/notices/unread-count | GET | 获取未读通知数量 |

## 七、部署说明

### 7.1 环境要求

-   **JDK**: 17 或更高版本
-   **Maven**: 3.6 或更高版本
-   **Node.js**: 18 或更高版本
-   **MySQL**: 8.0 或更高版本

### 7.2 快速部署步骤

1.  **导入数据库**: 执行 `database.sql` 文件创建数据库表结构
2.  **配置后端**: 修改 `application.yml` 中的数据库连接信息
3.  **启动后端**: 执行 `mvn spring-boot:run` 启动后端服务
4.  **安装前端依赖**: 在前端目录执行 `pnpm install`
5.  **启动前端**: 执行 `pnpm dev` 启动前端开发服务器
6.  **访问系统**: 浏览器访问 `http://localhost:3000`

详细的部署说明请参考 `README.md` 文件。

## 九、技术亮点

### 9.1 前端技术亮点

-   采用 Vue3 Composition API，代码组织更清晰，逻辑复用更方便
-   使用 Pinia 进行状态管理，替代 Vuex，API 更简洁
-   Vite 构建工具提供极速的热更新体验
-   Element Plus 组件库提供一致的 UI 体验
-   路由守卫实现自动登录状态检查

### 9.2 后端技术亮点

-   Spring Boot 3.x 提供最新的 Spring 生态支持
-   MyBatis-Plus 简化 CRUD 操作，自动填充创建时间和更新时间
-   JWT 无状态认证，支持分布式部署
-   统一异常处理和响应格式
-   分层架构设计，职责清晰，易于维护和扩展

### 9.3 安全性设计

-   密码采用 MD5 加密存储
-   JWT Token 有效期控制
-   接口统一鉴权拦截
-   细粒度的数据权限控制
-   CORS 跨域配置
